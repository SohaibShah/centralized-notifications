# AI Pizazz Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the AI features a scoped, on-brand visual identity — a vivid-cool gradient + sparkle on all AI surfaces, and a read-friendly bloom glow on the AI summary card.

**Architecture:** A small set of AI-scoped design tokens + keyframes + helper classes in `main.css` centralize the gradient/glow/bloom mechanics (matching the existing `.animate-enter` / `.toast-countdown` pattern). The three AI components then apply those classes: the AI summary card gets the gradient border + glow + click-bloom, the Ask AI tab and chat get the gradient text / sparkle / send-button identity.

**Tech Stack:** Vue 3 `<script setup>`, Tailwind v4 (`@theme` tokens + arbitrary values), `@lucide/vue`, Vitest.

## Global Constraints

- **Frontend only** — no backend/API/migration changes; branch `chore/qol-improvements`.
- **Palette (approved):** emerald-pine `#2f8060` → teal `#17a596` → sapphire `#3a72c8`, expressed as **OKLCH AI tokens** (verify in-browser against the hex; nudge L/C/H if off).
- **Glow (bloom) is AI-summary-card-only.** Ask AI tab + chat get gradient text / sparkle / send-button, **no** glow.
- **Bloom choreography:** rest ≈ 13% → hover ≈ 30% → click blooms to ≈ 55% then eases to rest over ~1.3s. One-shot.
- **Honor `prefers-reduced-motion`** — no drift/spin/bloom; glow renders static at rest opacity. (The app already has a global guard that zeroes `animation-duration`; the glow's _base_ opacity must be the rest value so it stays visible-but-static.)
- Do not change the AI summary's `aiOpen` expand/collapse, its `aria-expanded`/`aria-controls`, or the `settings.flags.aiSummaryEnabled` kill-switch.
- TS strict; `pnpm lint` + `pnpm typecheck` clean before a task is done. Conventional Commits; **never** add AI-attribution trailers.

/* ── AI-features identity: gradient text/border/glow (see the ai-pizazz design spec) ──

.ai-gradient-bg {

/* Soft AI glow — summary-card only. Rest opacity is the static state (reduced-motion safe). */
.ai-glow {
  position: absolute;
  inset: -35%;
  border-radius: 9999px;
  background: conic-gradient(
    from 0deg,
    var(--color-ai-1),
    var(--color-ai-2),
    var(--color-ai-3),

- [ ] **Step 3: Add `blooming` state + `toggleSummary`**

In `InboxTab.vue` `<script setup>`, next to `const aiOpen = ref(false);`, add:

```ts
// One-shot "bloom" on the AI summary glow when the card is clicked; cleared when the CSS
// animation ends (fires ~instantly under reduced motion, leaving the glow at its rest opacity).
const blooming = ref(false);
function toggleSummary(): void {
  aiOpen.value = !aiOpen.value;
  blooming.value = true;
}
```
